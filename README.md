Способы работы приложения

http://localhost:8080/ccwe/            - запуск собранного приложения, с реальными данными

http://localhost:8080/ccwe/tests/ui/   - прогон тестов с реальными данными

Запуск приложения для более удобной разработки (в этом режиме изменение js скриптов не требует пересборки прилоежния):

>> node node-server.js
(требуется предварительная установка модулей nodejs: connect и http-proxy)

http://localhost:3000/                 - запуск приложения с тестовыми данными (из файов в формате json) из папки data,
http://localhost:3000/tests/ui/        - запуск тестов с тестовыми данными (из файов в формате json) из папки data,

http://localhost:4000/web/             - запуск приложения с реальными данными, аякс запросы проксируются на адрес http://localhost:8080/


Структура приложения

app                     - основной код приложения
app\controller          - контроллеры (связывают бизнес логику и представления, следует избегать бизнес логики в самих контроллерах)
app\locale              - файлы локализации приложения, константы для строк
app\model               - модели
app\overrides           - переопределение поведения стандартных компонентов
app\services            - сервисы (описывают бизнес-процессы и сложную обработку)
app\store               - классы для описания аякс запросов и их связи с объектами бизнес-логики
app\ux                  - кастомные компоненты, не входящие во фреймворк ExtJs
app\view                - представления, следует избегать какой-либо бизнес логики в них, код должен быть максимально простым и декларативным

tests                   - тесты, модульные (фреймворк jsTestDriver) и функциональные (фреймворк Siesta)
tests\http              - тесты интеграционные для тестирования ответов от сервера
tests\ui                - тесты функциональные (фреймворк Siesta)
tests\unit              - тесты модульные (фреймворк jsTestDriver)
tests\jstestdriver      - служебные файлы для фреймворка jsTestDriver

css                     - файлы стилей
data                    - тестовые данные в формате json
resources               - сторонние библиотеки
jshint.json             - конфиг для статического анализа кода
jsTestDriver.conf       - конфиг для модульных тестов, должен располагаться в корне проекта для корректной подгрузки классов
node-server.js          - скрипт для запуска легкого веб-сервера для работы фронтенда

Сборка всего проекта выполняется командой (статический анализ кода и модульные тесты)

mvn clean jshint:lint dependency:copy-dependencies initialize integration-test


Стили
Стили разработаны с помощью технологии sass и организованны в виде темы оформления.
Тема находится в каталоге \packages\courier-online-theme
Для сборки темы необходим пакет утилит Sencha Cmd 4
Cборка стилей осуществляется командой
sencha app build

Сгенерированные скрипты попадают в каталог \build\production\COURIERONLINE\resources
Для обновления стилей их нужно скопировать в каталог resources.
